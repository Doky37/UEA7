## En este trimestre, es necesario contar con un repositorio general donde se puedan subir todos los códigos requeridos, como archivos HTML, CSS y JavaScript, que se soliciten en los trabajos y tareas a lo largo del ciclo. Tener un repositorio en GitHub permite gestionar, almacenar y compartir estos proyectos de manera organizada mediante el uso de control de versiones.
## El objetivo principal de utilizar GitHub es mantener tus códigos en un espacio seguro y accesible, además de llevar un historial completo de los cambios realizados. Esto facilita entender la evolución del proyecto, recuperar versiones anteriores si ocurre algún error y mantener un registro claro del progreso.
## Otro aspecto importante es la colaboración. GitHub permite que varias personas trabajen en el mismo proyecto sin riesgo de sobrescribir o perder trabajo. Herramientas como las ramas (branches) y los pull requests ayudan a organizar las contribuciones y revisar los cambios antes de integrarlos al proyecto principal. Además, al estar el repositorio almacenado en línea, es posible subir, actualizar y sincronizar archivos de manera automática, sin necesidad de hacerlo de forma manual desde la página, lo que agiliza el flujo de trabajo y hace más eficiente el desarrollo del proyecto.

## El repositorio tiene tres archivos, uno es de lenguaje HTML, otro es de lenguaje CSS y el tercero es de lenguaje JSON.
